MongoDB Shell :

The MongoDB Shell is the quickest way to connect to and work with MongoDB Database.

Commands :
1)help :To show all MongoDB Shell Command.
2)show dbs :Print a list of all current databases.
3)show db :Print a current database that you are working.
4)show collections :Print a list of all collection in the current database(like SHOW TABLES in MySQL command).
5)cls :Clear the Screen.
6)use :Set the current dataabase.
7)exit :Quit the MongoDB Shell.
8)quit :Quit the MongoDB Shell.

BSON(Binary Javascript Object Notation) :

  BSON is the data format that MongoDB uses to organize and store data.
  The BSON data format includes all JSON data structure types and adds support for types including dates, different size integers, and binary data.

JSON vs BSON

JSON BSON
1)Data Support String,Boolean,Number,Array,Object,null String,Boolean,Number(Integer,Float,Decimal),Array,Date,null,BinData
2)Readability Human and Machine Machine Only

Find in Database (like SELECT query in MySQL)

1)db.collectionName.find() : Returns all documents in the current database. (like select * from tableName in MySQL)

For Specific Queries (like WHERE clause in MySQL)
2)db.collectionName.findOne({key:value}) : Returns the first document that matched with the specified criteria.
3)db.collectionName.find({key:value,key:value,..}) : Returns the cursor(reference) of all documents that matched with the specified criteria.

Difference between find() and findOne()

find() findOne()
1)It return a cursor document(i.e reference of original document) 1)It return an actual/original document.

Query Operators for find in Database :

url search on google: Query Operators

Comparision Operator :
1)$eq Matches values that are equal to a specified value.
2)$gt Matches values that are greater than a specified value.
3)$gte Matches values that are greater than or equal to a specified value.
4)$in Matches any of the values specified in an array.
5)$lt Matches values that are less than a specified value.
6)$lte Matches values that are less than or equal to a specified value.
7)$ne Matches all values that are not equal to a specified value.
8)$nin Matches none of the values specified in an array.

Logical Operator : 1)$and Joins query clauses with a logical AND returns all documents that match the conditions of both clauses.
2)$or Joins query clauses with a logical OR returns all documents that match the conditions of either clause.
3)$not Inverts the effect of a query expression and returns documents that do not match the query expression.
4)$nor Joins query clauses with a logical NOR returns all documents that fail to match both clauses.

Update in Database (like UPDATE query in MySQL)

Update Operators :
1)$addFields :Adds new field to documents.
2)$set :Adds new field to documents.If new field is present then the $set operator replaces the value of a field with the specified value.

Update Methods :
1)db.collectionName.updateOne() :Updates at most a single document that match a specified filter even though multiple documents may match the specified filter.
2)db.collectionName.updateMany() :Update all documents that match a specified filter.
3)db.collectionName.replaceOne() :Replaces at most a single document that match a specified filter even though multiple documents may match the specified filter.

DELETE in Database (like DELETE query in MySQL)

Methods :

1)deleteOne() :
    Delete at most a single document that match a specified filter even though multiple documents may match the specified filter.
    Syntax : db.collectionName.deleteOne()
2)deleteMany() :
    Delete all documents that match a specified filter.
    Syntax : db.collectionName.deleteMany()
3)db.collectionName.deleteMany({}) :
    Delete all documents/records in that collection/table.
